# Architecture Decisions

> This file tracks significant technical decisions, their rationale, and trade-offs. Future agents should reference this before making similar choices.

## How to Add Decisions

When making an important choice:
1. Document the alternatives considered
2. Explain why you chose one over others
3. Note the trade-offs and limitations
4. Link to git commits if applicable

---

## Decision Template

```markdown
## Decision: [Title]
**Date:** YYYY-MM-DD
**Context:** [What problem are we solving?]
**Chosen:** [What we decided]
**Rejected:** [Alternatives not chosen]
**Rationale:**
- Reason 1
- Reason 2
**Trade-offs:**
- Pro: [Benefit]
- Con: [Limitation]
**Mitigation:** [How we handle the downsides]
**Related:** [Links to patterns.md, failures.md]
**Commit:** [Git SHA if applicable]
```
